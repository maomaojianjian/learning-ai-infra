## 1. 基础理论

### 1.1 CAP 理论

- **Consistency（一致性）**：所有节点在同一时刻数据一致
- **Availability（可用性）**：每个请求都能获得响应
- **Partition Tolerance（分区容错性）**：网络分区时系统仍能工作

> 三者不可兼得，AI 集群通常选择 AP（优先可用性 + 分区容错）。

### 1.2 BASE 理论

- **Basically Available**：基本可用
- **Soft State**：软状态
- **Eventually Consistent**：最终一致性

### 1.3 最终一致性 vs 强一致性

| | 强一致性 | 最终一致性 |
|--|---------|-----------|
| **延迟** | 高 | 低 |
| **可用性** | 低 | 高 |
| **适用** | 分布式锁/选主 | 状态同步/配置下发 |

## 3. 分布式共识

### 3.1 Raft 算法

- **选主（Leader Election）**：Term 递增、随机超时
- **日志复制（Log Replication）**：Leader 写入 → Follower 确认 → 提交
- **快照（Snapshot）**：压缩历史日志
- **脑裂防护**：Term 比较、旧 Leader 自动降级

### 3.2 Paxos 算法

- 更早的共识算法，Raft 是其工程化改进
- 更难理解，但理论更完整

### 3.3 etcd 实现

- 基于 Raft 的分布式 KV 存储
- AI 集群中用于：配置管理、选主、服务发现

## 5. 流量与容错

| 机制 | 说明 | AI 场景 |
|------|------|---------|
| **限流** | 令牌桶/漏桶算法 | 推理 API 限流 |
| **熔断** | 故障快速失败 | 模型服务降级 |
| **降级** | 服务降级策略 | 非关键功能关闭 |
| **重试** | 指数退避重试 | 训练任务失败重试 |
| **超时控制** | 层级超时设计 | 推理请求级联超时 |
| **幂等设计** | 防重复处理 | Checkpoint 保存 |

## 6. 负载均衡

- **四层/七层负载均衡**：L4（连接级）/ L7（请求级）
- **一致性哈希**：节点增减影响最小化
- **本地优先调度**：减少跨节点通信
- **亲和调度**：任务与数据亲和

## 7. AI 场景专属约束

- **分布式训练梯度同步一致性**：同步/异步训练权衡
- **Checkpoint 分布式读写一致性**：多节点并发读写锁
- **大规模集群任务分片**：任务分片容错
- **局部故障无感**：故障自愈设计
