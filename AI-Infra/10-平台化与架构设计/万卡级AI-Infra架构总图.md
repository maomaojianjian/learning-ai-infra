# 万卡级 AI Infra 整体架构总图

> 自上而下的八层架构设计，覆盖从业务接入到硬件资源的完整技术栈。

---

```
┌─────────────────────────────────────────────────────────────┐
│  业务接入层                                                  │

├─────────────────────────────────────────────────────────────┤
│  任务编排层                                                  │
│  Workflow 流水线、任务模板、环境管理、模型/数据集版本仓库       │
├─────────────────────────────────────────────────────────────┤
│  算力调度层                                                  │
│  K8s 集群底座 + Volcano 批量调度 + 多租户队列 + GPU 虚拟化     │
├─────────────────────────────────────────────────────────────┤
│  运行时层                                                    │
│  OCI 容器、AI 标准化镜像、CDI 设备注入、资源隔离、运行时 QoS   │
├─────────────────────────────────────────────────────────────┤
│  高速互联层                                                  │
│  四网物理隔离：管理网/存储网/训练 RDMA 网/推理业务网            │

│  存储底座层                                                  │
│  并行文件系统(训练) + S3 对象存储(模型/数据集) + 本地 NVMe 缓存 │
├─────────────────────────────────────────────────────────────┤

└─────────────────────────────────────────────────────────────┘
```

---

## 2. 核心业务依赖

| 业务场景 | 核心依赖 |
|----------|----------|
| **大模型训练** | 无损 RDMA 网络 + 并行文件系统 + NCCL 高性能通信 |
| **在线推理** | KV Cache 显存池化 + 动态扩缩容 + 热点模型本地缓存 |
| **多租户** | 命名空间隔离 + 队列优先级 + 存储/网络/权限强隔离 |

---

## 3. 万卡集群关键设计

### 3.1 算力层

- GPU 选型：训练用 H100/H200/B200，推理用 L40S/H100
- 节点规格：8×GPU/节点，SXM 形态（NVSwitch 互联）
- 故障域：机架级容灾（每机架 4 节点为故障域）

### 3.2 网络层

- 训练网：IB NDR 400Gbps / RoCE v2
- 存储网：100Gbps+ 以太网
- 拓扑：Fat-Tree / DragonFly

### 3.3 存储层

- 热层：Lustre/Weka 并行文件系统（TB/s 级吞吐）
- 温层：MinIO 对象存储
- 冷层：归档存储（自动生命周期）

### 3.4 调度层

- K8s + Volcano：Gang 调度、队列 QoS
- 优先级：在线推理 > 微调 > 离线训练 > 数据预处理
- 弹性：Spot 实例混部、闲时回收

---

> 🔑 **核心要点**：万卡集群的核心挑战不在单点性能，而在整体协同。八层架构的分层设计和三层业务依赖模型是架构评审的标准框架。
