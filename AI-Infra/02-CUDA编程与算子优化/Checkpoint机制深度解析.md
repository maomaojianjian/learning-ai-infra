# Checkpoint 机制深度解析

> 用**空间换时间**，在计算链路的中间保存部分中间结果（检查点），以减少需要长期保存的激活内存，同时在反向或后续计算时按需重新计算。

## 1. 核心定义

```
输入 x → f₁ → f₂ → ... → fₙ → 输出 y
          ↑保存   ↑保存        ↑保存
        检查点(Checkpoint)
```

| | 无检查点 | 有检查点 |
|--|---------|---------|
| **保存内容** | 保存所有中间激活 | 只保存少量检查点 |
| **内存** | 内存占用大 O(N) | 内存占用小 O(√N)~O(log N) |
| **计算** | 低 O(N) | 高 O(N·α)，α>1 |
| **复杂度** | 低 | 中等 |

---

## 2. 设计动机

长链路的反向传播需要保存大量中间激活，成为内存与效率的瓶颈。

**核心权衡**：减少内存占用 ⟷ 增加计算开销

---

## 3. 机制分析

以分段检查点为例（每 k 层保存一个检查点）：

- **前向（保存检查点）**：x → ① → ② → ... → Ⓚ → ... → Ⓝ，仅保存 C，丢弃中间激活
- **反向（按需重算）**：从 Cₘ₋₁ 重算到 Cₘ，利用结果反向传播

**微观视角**：
- Linear → GELU → Dropout → ...
- 每个算子是确定性的，输入相同则输出相同
- 无需保存中间激活，节省内存

---

## 4. 训练 vs 推理

| | 训练 (Training) | 推理 (Inference) |
|--|-----------------|------------------|
| **目标** | 反向传播更新参数 | 仅前向计算得到输出 |
| **检查点** | 使用检查点降低显存占用 | 通常不需要检查点 |
| **流程** | 输入 → 前向(保存检查点) → 损失 → 反向(重算+求梯度) → 更新 | 输入 → 前向(无需保存) → 输出 |
| **效果** | 显著降低显存，支持更长序列/更大batch | 追求最低延迟与最高吞吐 |

---

## 5. 常见策略

- **固定间隔分段**：均匀分段保存检查点
- **自适应分段**：根据内存压力动态调整
- **拓扑感知分段**：按耗时/内存拓扑感知保存
- **与混合精度、显存优化器配合使用**

---

> 🔑 **核心要点**：Checkpoint 是"空间换时间"的经典应用。在大模型训练中，配合 ZeRO、混合精度等技术，可显著降低显存占用，使更大 batch size 和更长序列的训练成为可能。
