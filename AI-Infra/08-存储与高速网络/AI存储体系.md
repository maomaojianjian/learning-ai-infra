## 1. 存储分层架构

### 1.1 三层存储模型

| 层级 | 存储类型 | 用途 | 性能要求 |
|------|----------|------|----------|
| **热层** | 并行文件系统(Lustre/Weka) | 训练读写加速 | 极高吞吐 |
| **温层** | 高性能对象存储(MinIO/S3) | 数据集、模型权重 | 高吞吐 |
| **冷层** | 归档存储 | 历史 Checkpoint、过期数据 | 低成本 |

## 2. 并行文件系统

大模型训练的核心存储依赖。

### 2.1 Lustre 架构

| 组件 | 功能 |
|------|------|
| **MDS** | 元数据服务器（Metadata Server） |
| **OSS** | 对象存储服务器（Object Storage Server） |
| **OST** | 对象存储目标（Object Storage Target） |
| **条带化** | 数据分布策略，并行读写 |

**优化策略**：
- 条带大小 = 1MB（适合大文件顺序读写）
- 元数据集群：多 MDS 并发处理
- 客户端挂载参数：`flock`、`localflock`

### 2.2 商用并行存储

- **Weka**：高性能并行文件系统
- **IBM Spectrum Scale（GPFS）**：企业级并行存储

### 2.3 优化策略

- 小文件优化：合并存储、元数据缓存
- 元数据瓶颈解决：多 MDS、元数据 SSD
- 并发读写锁控制
- 故障降级：OST 故障不影响集群

## 3. 对象存储

### 3.1 MinIO 集群架构

- **纠删码**：数据冗余保护（EC: N/2）

- **跨区域复制**：灾备

### 3.2 S3 协议兼容

- 模型权重/数据集版本化
- 预签名 URL、权限控制
- 生命周期策略自动分层

### 3.3 多级缓存

- 全局缓存 + 节点本地热点缓存

## 5. AI 专属存储痛点解决方案

### 5.1 海量小文件

- 数据集碎文件合并（WebDataset/TFRecord）
- 索引优化、元数据缓存
- 小文件打包大文件

### 5.2 大文件顺序读写

- 条带划分、预读策略
- IO 队列优化

### 5.3 Checkpoint 高频落盘

| 策略 | 说明 |
|------|------|
| **异步落盘** | 非阻塞保存，先到内存再刷盘 |
| **增量保存** | 只保存差异，减少写入量 |
| **压缩存储** | 减少存储占用 |
| **冷热自动迁移** | 自动分层，热数据留在高性能层 |

### 5.4 数据版本管理

- Dataset 版本、Model 版本
- 流水线数据快照

---

> 🔑 **核心要点**：存储是大模型训练的基础设施瓶颈之一。三层存储模型（热/温/冷）是架构设计的核心范式，并行文件系统的条带化策略和 Checkpoint 异步落盘是训练效率的关键。
