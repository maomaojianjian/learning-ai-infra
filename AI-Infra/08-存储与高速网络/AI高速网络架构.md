## 2. AI 集群高性能网络

### 2.1 四网物理隔离

| 网络类型 | 用途 | 技术要求 |
|----------|------|----------|
| **管理网** | 运维、监控、管控 | 普通以太网 |
| **存储网** | 并行文件系统、对象存储 | 高速以太网 |
| **训练网** | IB/RoCE 无损网络、NCCL 通信 | RDMA、无损 |
| **业务网** | 在线推理、对外服务 | 普通以太网 |

### 2.2 InfiniBand 架构

| 组件 | 功能 |
|------|------|
| **HCA 网卡** | 主机通道适配器（Host Channel Adapter） |
| **IB 交换机** | 高带宽低延迟交换 |
| **SM 子网管理器** | 子网管理（路由、分区） |
| **IB 分区** | 网络分区隔离 |

IB 速率演进：EDR(100G) → HDR(200G) → NDR(400G) → XDR(800G)

### 2.3 RoCE v2 无损网络

| 技术 | 说明 |
|------|------|
| **PFC** | 优先级流控制（Priority Flow Control），逐跳无损 |
| **ECN** | 显式拥塞通知（Explicit Congestion Notification） |
| **拥塞控制** | DCQCN 等无损拥塞控制算法 |
| **缓存分配** | 交换机 buffer 管理，防止丢包 |

### 2.4 IB vs RoCE

| | InfiniBand | RoCE v2 |
|--|-----------|---------|
| **延迟** | 极低（~1μs） | 低（~2-3μs） |
| **成本** | 高 | 中 |
| **生态** | NVIDIA/Mellanox | 多厂商 |
| **适用** | 超大规模训练 | 中小规模/混合部署 |

## 3. 集合通信网络优化

- **NCCL 拓扑感知**：拓扑文件配置
- **多网卡绑定**：带宽聚合
- **跨节点通信**：延迟优化、带宽瓶颈
- **网络抢占隔离**：训练/推理网络隔离

### 常用排查命令

```bash
ibstat               # IB 网卡状态
ibv_devinfo           # IB 设备详情
rdma link show        # RDMA 链路
nvidia-smi topo -m    # GPU 拓扑
```

---

## 4. 网络架构容灾

- 交换机堆叠、冗余组网
- 链路备份（多路径）
- 网络故障快速切换（< 50ms）

### 故障排查方法论

1. 检查物理链路：`ibstat` / 光模块功率
2. 检查 RDMA 连通性：`ib_send_bw` 带宽测试
3. 检查 NCCL 通信：`NCCL_DEBUG=INFO` 日志
4. 检查交换机配置：PFC/ECN 参数、buffer 分配

---

> 🔑 **核心要点**：网络是分布式训练的通信生命线。四网物理隔离是架构基础，RDMA（IB/RoCE）是性能保障，NCCL 拓扑感知是优化关键。网络故障排查需要从物理层→链路层→应用层逐层定位。
