## 🎯 各模块导读

### 一、前置知识与硬件基础

AI Infra 的底层是硬件算力。理解 GPU 架构、显存层级、互联拓扑是后续一切优化的前提。

- **NVIDIA Hopper 架构深度解析**：H100 的第四代 Tensor Core、Transformer Engine、NVLink 4.0、HBM3 等关键特性
- **GPU 硬件演进**：从 Ampere 到 Hopper 再到 Blackwell 的架构差异
- **CUDA 编程模型基础**：线程层级、内存模型、Kernel 启动

📂 [进入模块 →](./01-前置知识与硬件基础/)

---

### 二、CUDA 编程与算子优化

写出能跑的 Kernel 只是第一步，写出跑得快的 Kernel 才是核心价值。

- **LeetCUDA**：200+ 分级 CUDA Kernel（Easy → Hard++），涵盖 Tensor Core / CUDA Core、HGEMM、FlashAttention 实现，配套 100+ 技术博客
- **算子优化核心**：Tiling、Memory Coalescing、Bank Conflict 消除、Warp Shuffle
- **性能分析**：Roofline 模型、Nsight Systems/Compute

📂 [进入模块 →](./02-CUDA编程与算子优化/)

---

### 三、分布式训练

大模型训练必须依赖分布式。本模块覆盖从通信原语到训练框架的完整链路。

| 项目 | 定位 | 核心技术 |
|------|------|----------|
| **NCCL** | GPU 通信基座 | AllReduce/AllGather/Broadcast，Ring/Tree 算法 |
| **DeepSpeed** | 训练优化框架 | ZeRO/ZeRO-Infinity/Offload，3D 并行，Ulysses 序列并行 |
| **Megatron-LM** | 大规模训练框架 | TP/PP/DP/EP/CP，FP8/FP4，最高 462B 参数训练 |
| **DeepEP** | MoE 通信库 | 专家并行 all-to-all，FP8 dispatch/BF16 combine，0-SM 实验特性 |

📂 [进入模块 →](./03-分布式训练/)

## 🗺️ 学习路线建议

参考 AIInfraGuide 的模块化设计，建议按以下路径学习：

```
阶段1：筑基期（1-3个月）
  ├── GPU 硬件基础（Hopper 架构、Tensor Core、显存层级）
  ├── Linux 内核调优（CPU/内存/网络/IO 子系统）
  ├── CUDA 编程入门（LeetCUDA Easy/Medium）
  ├── PyTorch 框架与自动微分
  └── 分布式系统基础理论（CAP/Raft/Paxos）

阶段2：云原生 + AI 调度（3-6个月）
  ├── K8s 核心组件深度（apiserver/scheduler/etcd）
  ├── AI 专属扩展（Device Plugin、GPU 调度、Volcano）
  ├── NCCL 通信原理与 RDMA 网络
  └── 并行文件系统与对象存储

阶段3：算子优化 + 编译器（3-6个月）
  ├── CUDA 性能优化（Tiling、Shared Memory、Warp 调度）
  ├── 经典算子实现（Reduce、GEMM、Softmax、FlashAttention）
  ├── Triton 算子开发与编译器架构
  └── Checkpoint 机制与 AI 编译器专题

阶段4：训推全链路（6-9个月）
  ├── DeepSpeed ZeRO / Megatron-LM 3D/4D/5D 并行
  ├── vLLM / SGLang / llama.cpp 部署与调优
  ├── 量化（INT8/FP8/INT4）与 Speculative Decoding
  └── 训练稳定性与故障排查

阶段5：高阶架构（9-12个月）
  ├── 存储分层架构设计
  ├── RoCE/IB 无损网络部署与优化
  ├── 可观测全栈（Prometheus/Grafana/OpenTelemetry）
  └── 成本治理与安全架构

阶段6：架构师专家能力（12-15个月）
  ├── 万卡集群架构设计（八层分层架构）
  ├── 开源项目二次开发（vLLM/NCCL/Volcano）
  ├── 平台化设计（自研 AI 平台）
  ├── 容灾安全体系
  └── 答辩晋升与全局架构评审
```
