# 架构师答辩与晋升话术

> AI Infra 架构师晋升/答辩的标准话术框架，涵盖自我定位、方案设计、问题定位、降本增效等核心场景。

---

## 1. 自我定位话术

> 深耕 AI Infra 全栈，具备算力-网络-存储-调度-训推-可观测-成本-安全全链路架构设计与落地能力，可独立完成从硬件选型、集群搭建、平台设计到性能调优、稳定性治理、多租户运营的完整闭环，单人可覆盖传统集群、容器化 AI 平台、大模型训推一体化全部研发与运维工作，具备万卡级集群架构演进设计能力。

---

## 2. 方案设计标准话术（六要素）

| 要素 | 话术 |
|------|------|
| **现状痛点** | 资源利用率低、环境碎片化、通信性能瓶颈、存储 IO 瓶颈、运维成本高 |
| **架构目标** | 标准化、可扩展、高可用、高性能、低成本、强隔离 |
| **分层设计** | 自上而下分层解耦，单一职责、横向扩展、故障域隔离 |
| **选型原则** | 业务适配、社区成熟、可二次开发、可平滑扩容 |
| **容灾稳定性** | 多副本、故障隔离、自动恢复、全链路监控告警 |
| **成本优化** | 资源混部、闲时复用、冷热分层、弹性伸缩 |

## 3. 问题定位通用逻辑

### 问瓶颈

> 从「硬件 → 网络 → 存储 → 调度 → 应用」逐层拆解定位。先排除最底层（硬件/网络），再向上排查（调度/应用层）。

排查顺序：
```
GPU 掉卡？ → 网卡故障？ → 存储 IO 瓶颈？ → 调度排队？ → 代码 Bug？
```

### 问高可用

> 故障域划分 + 冗余 + 自动转移 + 降级兜底。

- 单 GPU 故障：任务自动重调度到健康节点
- 单节点故障：Checkpoint 续训，≤ 5 分钟恢复
- 机房级故障：跨机房容灾（冷备）

### 问降本

> 利用率提升 + 错峰混部 + 存储分层 + 弹性资源。

- 利用率：50% → 80%（混部 + 碎片清理）
- 存储：冷热分层，年省 30%
- 算力：Spot 实例混部，年省 40%

### 问隔离

> 资源隔离 + 网络隔离 + 权限隔离 + 租户配额。

- 硬隔离：MIG 切分、独立网络 VLAN
- 软隔离：Namespace 配额、RBAC

## 4. 答辩高频问题准备

| 问题 | 回答框架 |
|------|---------|
| **训练慢怎么办？** | 先看 GPU 利用率（是否 < 80%）→ 再看 NCCL 通信带宽（是否达理论 90%+）→ 再看数据加载（是否有 IO 等待） |
| **集群不稳定怎么排查？** | 分层定位：硬件层（GPU/网卡健康）→ 网络层（RDMA 丢包）→ 调度层（资源竞争）→ 应用层（代码 Bug） |
| **如何设计万卡集群？** | 分八层设计（接入→编排→调度→运行时→互联→存储→硬件→可观测），每层独立扩缩容、独立容灾 |
| **怎么保证推理 SLA？** | 独立 GPU 不抢占 + 多副本 + 自动扩缩容 + P99 延迟告警 |
| **如何选型 GPU？** | 训练选 H100/H200/B200（高显存带宽），推理选 L40S/H100（性价比），小模型推理可选 A10/L4 |

---

## 5. 价值呈现技巧

- **量化表达**：GPU 利用率提升 X%、延迟降低 Xms、成本节省 X 万/年
- **横向对比**：vs 业界标杆（vLLM PagedAttention、Megatron 3D 并行）
- **可复用性**：方案是否可推广到其他团队/业务线
- **技术影响力**：是否推动开源贡献、内部技术分享、标准制定

---

> 🔑 **核心要点**：架构师答辩的核心不是展示技术深度，而是展示**全局决策能力**和**价值创造能力**。所有回答都要落到"带来了什么业务价值"上——性能提升、成本降低、稳定性增强。
