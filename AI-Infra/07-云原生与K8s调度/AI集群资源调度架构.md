## 1. 任务模型区分

| 任务类型 | SLA | 资源诉求 | 优先级 | 混部策略 |
|----------|-----|----------|--------|----------|
| **离线训练** | 低 | 高 GPU | 中 | 可抢占 |
| **微调任务** | 中 | 中 GPU | 中高 | 可等待 |
| **在线推理** | 极高 | 中 GPU | 最高 | 不可抢占 |
| **数据预处理** | 低 | 高 CPU/IO | 低 | 闲时运行 |
| **调试开发** | 中 | 低 GPU | 低 | 弹性分配 |

## 2. GPU 调度精细化方案

| 模式 | 适用场景 | 隔离级别 |
|------|----------|----------|
| **独占 GPU** | 高优先级大模型预训练 | 物理隔离（最强） |
| **共享 GPU** | 微调/小模型推理 | MPS/显存限制隔离 |
| **虚拟化 GPU** | MIG/vGPU 多租户 | 硬隔离 |
| **异构混部** | CPU + GPU + IO 密集型 | 错峰调度 |

核心策略：
- 离线训练可抢占 → 提高整体利用率
- 在线推理不可抢占 → 保障 SLA
- 闲时复用资源 → 降本增效

## 3. 多租户架构设计

### 3.1 租户隔离维度

| 维度 | 实现方式 |
|------|----------|
| **命名空间隔离** | K8s Namespace |
| **资源配额** | CPU/内存/GPU/存储配额 |
| **网络隔离** | VLAN/VXLAN 隔离 |
| **存储隔离** | 独立存储桶/目录 |
| **权限隔离** | RBAC 最小权限 |

### 3.2 租户层级

```
部门 → 项目 → 用户
```

三级权限体系：
- **资源 QoS**：优先级队列
- **资源抢占阈值**：高优先级可抢占低优先级
- **闲时资源共享**：低优任务使用空闲资源

## 4. 弹性与扩缩容

- **集群自动扩缩容**：节点自动上下线
- **抢占式实例混合使用**：成本优化（Spot/Preemptible）
- **任务弹性**：
  - 弹性队列
  - 突发算力资源池
  - 闲时资源回收

### 扩缩容策略

```
高负载 → 自动扩容（新增 GPU 节点）
低负载 → 自动缩容（回收空闲节点）
训练任务排队 → 弹性提供临时算力
推理高峰 → 提前预留 buffer
```

---

> 🔑 **核心要点**：调度架构的核心是在**利用率**和**SLA**之间找到平衡。Gang 调度保障训练任务原子性，多租户隔离保障安全，弹性扩缩降本增效。
