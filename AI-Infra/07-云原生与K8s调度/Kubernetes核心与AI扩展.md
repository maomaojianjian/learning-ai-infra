## 1. Kubernetes 核心原理

### 1.1 控制面组件

| 组件 | 功能 |
|------|------|
| **kube-apiserver** | API 入口、认证授权、准入控制 |
| **etcd** | 分布式键值存储、数据一致性 |
| **kube-controller-manager** | 节点/副本/端点控制器 |
| **kube-scheduler** | 默认调度器、预选/优选策略 |
| **cloud-controller-manager** | 云厂商接口 |

### 1.2 数据面组件

| 组件 | 功能 |
|------|------|
| **kubelet** | 节点代理、Pod 生命周期管理 |
| **kube-proxy** | 服务代理、负载均衡 |
| **容器运行时** | containerd、CRI-O |

### 1.3 资源模型

- **Pod**：最小调度单元、多容器共享
- **Node**：节点资源、标签/污点
- **Namespace**：资源隔离、多租户基础
- **Quota/LimitRange**：资源配额/限制
- **Taint/Toleration**：污点容忍、节点筛选
- **NodeAffinity/PodAffinity**：亲和/反亲和调度

### 1.4 控制器模式

- **Reconcile 循环**：期望状态与实际状态对齐
- **CRD**：自定义资源定义
- **Operator 开发范式**：自定义控制器

## 2. AI 专属 K8s 扩展

### 2.1 Device Plugin

- **nvidia-device-plugin**：GPU 资源上报
- **异构设备插件**：NPU/MLU 设备插件开发规范
- **CDI 设备注入标准**：容器设备接口（Container Device Interface）

### 2.2 容器运行时

- **containerd**：全配置、OCI 标准
- **nvidia-container-runtime**：GPU 容器运行时
- **runtime handler**：多运行时支持

### 2.3 GPU 资源调度扩展

| 模式 | 说明 | 隔离级别 |
|------|------|----------|
| **整卡 GPU** | 资源上报与调度 | 物理隔离 |
| **显存分片 GPU** | 细粒度 GPU 调度 | 显存隔离 |
| **MIG 子设备** | 多实例 GPU 调度 | 硬件隔离 |
| **共享 GPU 配额隔离** | 显存限制、算力限制 | 软隔离 |

## 3. 调度体系深度

### 3.1 默认调度器

- **预选策略**：硬性条件过滤
- **优选策略**：软性条件打分
- **资源打分**：多维度评分
- **抢占机制**：高优先级抢占低优先级
- **驱逐机制**：资源不足时 Pod 驱逐

### 3.2 批量调度定制 — Volcano

Volcano 是 CNCF 的云原生批量调度引擎，专为 AI/大数据设计：

**核心架构**：
| 概念 | 说明 |
|------|------|
| **Queue** | 队列管理，支持多级队列 |
| **Job** | 作业定义（训练/推理任务） |
| **TaskGroup** | 任务组，支持 Gang 调度 |
| **优先级** | 任务优先级、抢占、队列分级 |

**对比传统调度**：
| 调度器 | 优势 | 适用 |
|--------|------|------|
| **Volcano** | Gang 调度、队列QoS、公平调度 | AI 训练 |
| **YARN** | 成熟生态、大数据集成 | 混合集群 |
| **Slurm** | HPC 传统、低延迟 | 科研计算 |

### 3.3 调度架构设计

- **强抢占/弱抢占**：策略选择
- **队列层级**：多级队列设计
- **多租户资源隔离**：命名空间隔离
- **资源预留**：保障关键任务
- **训练任务 Gang 调度**：多卡任务整体调度、防止拆分配置
