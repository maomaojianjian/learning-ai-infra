## 1. Node.js 安装

### 1.1 免安装版配置

假设 Node.js 免安装版路径：

```
C:\Users\<用户名>\Downloads\node-v25.9.0-win-x64\node-v25.9.0-win-x64
```

### 1.2 添加到系统 PATH

以管理员身份打开 PowerShell：

```powershell
$targetPath = "C:\Users\<用户名>\Downloads\node-v25.9.0-win-x64\node-v25.9.0-win-x64"
$currentPath = [Environment]::GetEnvironmentVariable("Path", "Machine")

if ($currentPath -split ';' -contains $targetPath) {
    Write-Host "Already in PATH"
} else {
    [Environment]::SetEnvironmentVariable("Path", "$currentPath;$targetPath", "Machine")
    Write-Host "Node.js path added to system PATH"
}
```

### 1.3 验证

```bash
node --version    # v25.9.0
npm --version     # 11.12.1
```

## 2. Claude Code 安装

### 2.1 安装命令

```bash
npm install -g @anthropic-ai/claude-code
```

### 2.2 解决 `command not found`

**原因**：npm 全局安装的可执行文件在 `C:\Users\<用户名>\AppData\Roaming\npm`，未加入 PATH。

**解决**：以管理员身份执行：

```powershell
$npmGlobalPath = "C:\Users\<用户名>\AppData\Roaming\npm"
$currentPath = [Environment]::GetEnvironmentVariable("Path", "Machine")

if ($currentPath -split ';' -contains $npmGlobalPath) {
    Write-Host "Already in PATH"
} else {
    [Environment]::SetEnvironmentVariable("Path", "$currentPath;$npmGlobalPath", "Machine")
    Write-Host "Added npm global path to system PATH"
}
```

**注意**：修改后需重启终端生效。

## 3. Git Bash 代理配置

### 3.1 前提条件

- 已安装 Git Bash
- 本地运行 Clash（或其他代理），记住端口号（示例为 `7897`）

### 3.2 永久代理配置

编辑 `~/.bashrc`：

```bash
export http_proxy=http://127.0.0.1:7897
export https_proxy=http://127.0.0.1:7897
export all_proxy=socks5://127.0.0.1:7897
```

生效：

```bash
source ~/.bashrc
```

### 3.3 验证代理

```bash
curl -I https://www.google.com
```

## 4. 常见问题排查

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| `node: command not found` | Node.js 未加入 PATH | 按 1.2 步骤添加 |
| `claude: command not found` | npm 全局目录未加入 PATH | 按 2.2 步骤添加 |
| Git Bash 无法翻墙 | 代理未配置 | 按 3.2 配置 ~/.bashrc |
| PowerShell 执行策略限制 | ExecutionPolicy 限制 | `Set-ExecutionPolicy RemoteSigned` |
