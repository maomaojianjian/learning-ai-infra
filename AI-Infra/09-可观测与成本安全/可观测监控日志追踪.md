## 1. 监控体系全栈

### 1.1 指标体系

| 维度 | 指标 |
|------|------|
| **硬件指标** | GPU 利用率、显存、温度、功耗、NCCL 通信吞吐 |
| **集群指标** | 节点负载、CPU/内存、网络 IO、磁盘 IO |
| **业务指标** | 训练吞吐、迭代耗时、推理 QPS/P99 延迟 |

### 1.2 监控组件

- **Prometheus/VictoriaMetrics**：时序数据库
- **Grafana**：可视化 Dashboard
- **自定义 Exporter 开发**：GPU/业务指标采集
  - DCGM Exporter（NVIDIA 官方 GPU 指标）
  - 自定义训练指标 Exporter

### 1.3 告警体系

| 层级 | 说明 |
|------|------|
| **严重** | GPU 掉卡、NCCL 超时、推理服务不可用 |
| **警告** | GPU 温度过高、显存使用 > 90%、P99 延迟上升 |
| **提示** | 训练 ETA 变化、存储使用率增长 |

**告警治理**：
- **告警降噪**：减少误报（聚合窗口、重复抑制）
- **告警聚合**：同类合并、根因定位
- **值班联动**：自动通知（钉钉/飞书/PagerDuty）

## 2. 日志系统

| 组件 | 功能 |
|------|------|
| **容器日志收集** | Fluentd/Fluent Bit sidecar |
| **日志结构化** | JSON 格式，统一 schema |
| **日志检索** | Elasticsearch/Loki + Grafana |
| **日志生命周期** | 归档/清理策略（热数据 7 天 → 冷归档 90 天） |

## 3. 链路追踪

- **OpenTelemetry**：标准化追踪协议
- **分布式追踪**：训练/推理全链路
  - 训练数据加载 → 前向 → 反向 → 梯度同步 → 参数更新
  - 推理请求 → 网关 → 调度 → 推理引擎 → 返回
- **调用链排查**：性能瓶颈定位

## 4. 故障演练与稳定性

### 4.1 混沌工程

- **故障注入**：随机杀 Pod、断网、磁盘满
- **容灾演练**：定期演练（季度/月度）
- **SLO/SLA 体系**：服务等级目标与协议
  - 训练可用性 > 99.5%
  - 推理可用性 > 99.9%

### 4.2 稳定性治理

| 实践 | 说明 |
|------|------|
| **故障复盘** | 5-Why 分析、改进措施跟踪 |
| **变更管控** | 灰度发布、可回滚、变更窗口 |
| **容量规划** | 提前扩容、资源预留 |
| **Runbook** | 标准化故障处理手册 |

---

> 🔑 **核心要点**：可观测体系的三大支柱是 Metrics（指标）、Logging（日志）、Tracing（链路追踪）。GPU 集群的监控除了常规指标外，还需特别关注 NCCL 通信带宽、GPU 显存碎片率、训练 Loss 异常等 AI 特有指标。
