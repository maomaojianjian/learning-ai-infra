# 分布式训练算法路径（五大并行维度）

> 纯 PyTorch 从零实现 5 大类并行算法（DP、TP、PP、CP、EP），不依赖 DeepSpeed 和 Megatron，硬核手撕关键算法。

---

## 1. 并行策略总览

| 维度 | 全称 | 关键算法/技术点 |
|------|------|----------------|
| **DP** | Data Parallelism | 分布式数据、DP 梯度、分布式 Adam、ZeRO-1/2/3、混合精度 Adam、IO-load-save |
| **TP** | Tensor Parallelism | col-parallel、row-parallel、SwiGLU、GQA、LMhead、Embedding、RMS-Norm、CrossEntropy |
| **PP** | Pipeline Parallelism | 梯度检查点、PP-basic、Gpipe、PipeDream、Zero-Bubble、Easy-Dualpipe、DualPipe |
| **CP** | Context Parallelism | Ring-AllReduce、Softmax、Online-Softmax、Ring-Softmax、FlashAttention-2、Ring-Attention |
| **EP** | Expert Parallelism | All2All、异步 All2All、TopK 梯度、SMoE、GShard、1F1B Basic、1F1B 重叠、V3-MoE 结构 |

---

## 2. 项目特点

- **纯 PyTorch 从零实现**：5 大类并行算法，不依赖 DeepSpeed 和 Megatron 框架
- **硬核手撕关键算法**：
  - Backward 手写实现
  - 分布式 gradient 和 ZeRO-ADAM
  - MoE EP 1F1B 下的通信-计算重叠
- **Step-by-step 手撕重点**：
  - DP: ZeRO-3
  - TP: Llama
  - CP: RingAttention
  - PP: DualPipe
  - EP: Gshard
- **低门槛运行**：不需要多卡环境，纯 CPU GLOO backend 可运行所有实例，无须 Triton 和 CUDA 等基础

---

## 3. 代码示例节选

### 3.1 GQA（Grouped Query Attention）

分组查询注意力机制，支持 TP 切分优化。

### 3.2 DualPipe

双向流水线调度，减少气泡率，实现通信-计算重叠。

### 3.3 RingAttention

环形序列并行，支持超长上下文训练。

### 3.4 MoE-EP

混合专家模型的专家并行，含 AllToAll 通信和 1F1B 调度。

---

> 🔑 **核心要点**：理解分布式训练的最佳方式是手写实现。从 DP 的 ZeRO-3 到 CP 的 RingAttention，再到 EP 的 AllToAll 通信，逐步深入理解每种并行策略的通信模式和计算逻辑。
